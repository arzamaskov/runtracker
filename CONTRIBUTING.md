# Contributing to Runtracker

Thank you for your interest in contributing to Runtracker.
We welcome contributions from the community and appreciate your help in making
the project better.

## Ways to Contribute

You can contribute by:
- reporting bugs
- suggesting features or improvements
- improving documentation
- submitting pull requests

## Development Workflow

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes with clear, focused commits
4. Ensure the project builds and tests pass
5. Open a Pull Request with a clear description of your changes

## Code Style and Quality

- Follow existing code style and conventions
- Keep changes small and focused
- Add tests when applicable
- Avoid unrelated refactoring

## Commit Messages

We use **Conventional Commits**:

```
<type>(scope): short description
```

Examples:
- `feat(training): add weekly mileage calculation`
- `fix(api): correct pace calculation`
- `docs(readme): update installation instructions`

## Licensing and CLA

By submitting a pull request or any other contribution, you agree that your
contribution is licensed under the **Apache License 2.0** and that you accept
the terms of the [Contributor License Agreement](./CLA.md).

If you are contributing on behalf of your employer, please ensure that you have
the legal right to do so.

## Code of Conduct

Be respectful and constructive.
Harassment, discrimination, or toxic behavior will not be tolerated.
