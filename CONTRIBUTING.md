# Contributing to Runtracker

Thank you for your interest in Runtracker!
This project exists thanks to people like you.

---

## Project Philosophy

Runtracker is:
- **Open source** (AGPL-3.0)
- **Self-host friendly**
- Built for **recreational runners**, not enterprise complexity

You are free to:
- run Runtracker locally or on your own server,
- modify it,
- share improvements â€” as long as AGPL terms are respected.

The official hosted version is paid and helps fund development.

---

## How You Can Contribute

We welcome:
- bug fixes,
- performance improvements,
- documentation,
- tests,
- UI/UX improvements,
- discussions and ideas.

Before starting large changes, please open an issue to discuss them.

---

## Development Principles

Please try to follow these principles:

- Keep things **simple**
- Prefer **readability over cleverness**
- Avoid unnecessary dependencies
- Backend-first, minimal frontend
- SQLite-friendly by default

---

## License & CLA

- The project is licensed under **GNU AGPL v3.0**
- By submitting a pull request, you agree to the terms of the [Contributor License Agreement (CLA)](CLA.md)

If you are contributing on behalf of an employer, make sure you are allowed to do so.

---

## Pull Request Guidelines

- One logical change per PR
- Clear commit messages
- Add or update tests when applicable
- Describe **why** the change is needed, not only *what* it does

Draft PRs are welcome.

---

## Code of Conduct

Be respectful.
We want this to be a friendly and inclusive project.

---

## Questions?

If anything is unclear:
- open an issue,
- or start a discussion.

We are happy to help.
